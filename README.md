The primary goal of this project is to showcase practical proficiency in the TDD cycle:

* **Red (Failing Test First):** Every new feature begins with a test that is guaranteed to fail before any implementation code is written.

* **Green (Minimal Code):** Writing only the minimal code necessary to make the failing test pass.

* **Refactor (Improve Code Safely):** Improving the design, structure, and clarity of the code while all tests remain green.
